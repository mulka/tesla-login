Try passkey registration and authentication right in your browser

💡 This demo runs entirely in your browser — no server needed. Credentials are created using your device's built-in authenticator (Touch ID, Face ID, Windows Hello, security key, etc.) and stored locally in memory for the session.

3 What Just Happened?

A breakdown of the WebAuthn flow you just performed.

During Registration

  1. The "server" generated a random challenge (32 random bytes)
  2. Your browser called the WebAuthn API with the challenge and user info
  3. Your device's authenticator created a new key pair
  4. You approved with biometrics or PIN
  5. The authenticator returned the public key and a signed attestation
  6. The "server" stored the credential ID and public key

During Authentication

  1. The "server" generated a new random challenge
  2. Your browser called WebAuthn with the challenge and the allowed credential ID
  3. Your authenticator found the matching private key
  4. You approved with biometrics or PIN
  5. The authenticator signed the challenge with the private key
  6. The "server" verified the signature using the stored public key

🔑 The private key never left your device. The server only ever sees the public key and signed challenges. Even if the server is breached, attackers can't impersonate you — they'd need your physical device and your biometric.
